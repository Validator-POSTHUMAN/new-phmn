# Liquidity SubDAO: Создание пула ликвидности и выкуп PHMN

## Контекст

Liquidity SubDAO выступает одновременно в двух ролях:

1. **Поставщик ликвидности** — создаёт пул PHMN/USDC на Osmosis DEX, внося обе стороны пула
2. **Покупатель PHMN** — использует дополнительный USDC для выкупа PHMN из этого же пула

Параметры пула и объём выкупа подобраны так, чтобы обеспечить целевое
соотношение активов в пуле после завершения операции.

| Параметр | Значение |
|----------|----------|
| PHMN в пуле (начальный) | 27,313 |
| USDC в пуле (начальный) | 11,624 |
| Swap fee | 0% |
| USDC на выкуп PHMN | 11,526 |
| Тип пула | Balancer 50/50 |
| Модель AMM | Constant Product: `x · y = k` |

## Расчёт выкупа

### Модель: Constant Product AMM (x · y = k)

Пул Osmosis типа Balancer с равными весами (50/50) использует формулу:

```
x · y = k
```

где `x` — количество PHMN, `y` — количество USDC, `k` — константа.

### Шаг 1: Константа пула до сделки

```
k = PHMN_initial × USDC_initial
k = 27,313 × 11,624
k = 317,486,312
```

### Шаг 2: Новый баланс USDC после внесения покупателем

```
USDC_after = USDC_initial + USDC_swap_in
USDC_after = 11,624 + 11,526 = 23,150
```

### Шаг 3: Новый баланс PHMN (из сохранения константы)

```
PHMN_after = k / USDC_after
PHMN_after = 317,486,312 / 23,150
PHMN_after ≈ 13,714.3115
```

### Шаг 4: Количество PHMN, полученное покупателем

```
PHMN_received = PHMN_initial − PHMN_after
PHMN_received = 27,313 − 13,714.3115
PHMN_received ≈ 13,598.6885
```

## Экономический смысл

### Цели операции

Операция направлена на решение последствий **несанкционированного минта избыточного
PHMN**. Конкретные задачи:

1. **Минимизация потерь держателей PHMN** — выкуп избыточного предложения с рынка
   снижает давление продаж и восстанавливает баланс токена.

2. **Предотвращение MEV-атак** — проведение выкупа через собственный пул ликвидности
   исключает возможность фронтраннинга и сэндвич-атак ботами, которые неизбежны
   при крупных рыночных ордерах в публичных пулах.

3. **Защита от спекуляций с использованием ботов и скриптов** — контролируемая
   среда собственного пула не позволяет автоматизированным системам извлекать
   выгоду за счёт держателей PHMN.

4. **Защита интересов держателей PHMN** — все действия выполняются Liquidity SubDAO
   в соответствии с мандатом, полученным от управления PHMN.

### Механика

1. **Создание пула с контролируемой ликвидностью** — Liquidity SubDAO единолично
   формирует пул, что даёт полный контроль над параметрами и исключает внешнее
   влияние на этапе выкупа.

2. **Выкуп крупным объёмом** (11,526 USDC — почти столько же, сколько весь USDC в пуле)
   обеспечивает изъятие значительного количества PHMN из обращения за одну сделку,
   без проскальзывания в пользу третьих лиц.

3. **После выкупа соотношение в пуле** (13,714 PHMN / 23,150 USDC) формирует
   новую рыночную цену PHMN ≈ 1.69 USDC на Osmosis DEX.

### Дальнейшая судьба выкупленного PHMN

Выкупленные **13,598.69 PHMN** направляются в **трежери Liquidity SubDAO**
и будут использованы **исключительно** для:

- Создания новых пулов ликвидности с PHMN
- Любых других решений, за которые проголосуют держатели PHMN, локающие токены
  в **POSTHUMAN DAS**

## Почему swap fee = 0

Liquidity SubDAO является **единственным поставщиком ликвидности** в пуле
(владеет 100% LP-токенов). В Osmosis, как и в других AMM, комиссия со свопа
распределяется между держателями LP-токенов пропорционально их доле.

Поскольку все LP-токены принадлежат Liquidity SubDAO:

- **Любая swap fee, уплаченная при свопе, немедленно возвращается тому же
  Liquidity SubDAO** в виде доли комиссий, начисляемых на LP-позицию.
- Для единственного LP комиссия — это перемещение средств из одного кармана
  в другой, не влияющее на итоговый баланс.

Таким образом, для расчётов **swap fee эффективно равна 0 при любом
номинальном значении**, и формула `x·y = k` применяется без корректировок.
